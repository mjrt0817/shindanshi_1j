# 品質改善・弱点自動出題・オフライン対応

更新日：2026-07-21

## 今回の主な改善

- 全838問を起動時に自動監査し、品質スコアと指摘事項を付与
- 品質基準未満・重大不備の問題を通常出題から除外
- 選択肢の意味重複、正解不整合、長さ偏り、極端表現、解説不足を検査
- 明確な品質問題があった高重要度10問の選択肢・解説を個別修正
- 解答前に「確信・迷い・勘」を記録
- 確信度を確定する前なら、選択肢の変更・同じ選択肢の再押下による解除が可能
- 正解でも「迷い・勘」は復習対象へ登録
- 確信して2回連続正解すると復習対象から除外
- 誤答・不確実正解・経過日数・未出題・重要度から「今日の30問」を自動構成
- 論点別の習熟度と弱点論点をホームに表示
- 品質監査結果をCSVで出力可能

## オフライン学習

VercelなどHTTPSで公開した場合、最初にオンラインで1回開くと、アプリ本体と問題データが端末へ保存されます。以後は通信がない状態でも、問題演習、確信度入力、誤答復習、履歴保存を利用できます。

- オフライン中の履歴はブラウザのlocalStorageへ保存されます。
- GoogleログインとFirestore同期はオフライン中は利用できません。
- 接続が戻ると、端末内の履歴をクラウドへ再同期します。
- 初回アクセス前に完全オフラインの場合は公開URLを開けません。
- `index.html`を端末へ保存して直接開く場合は、Firebase同期なしのローカル学習として利用できます。

## 公開時のファイル

`index.html`だけでなく、次のファイルも同じ階層へ配置してください。

- `service-worker.js`
- `manifest.webmanifest`
- `vercel.json`

オフライン対応を更新したときは、一度オンラインでアプリを開き直してください。
